各店の詳細ページから営業時間も取得済み（50/78 件）。
Hisho グルメ機能（`app/src/data/shops.ts`）の元データとして使う。

- 取得日: 2026-06-07
- 取得元: Google マップ共有リスト + 各店詳細ページ（browser-use-terminal で自動抽出）
- ファイル:

  - [`fukuoka-city-want-to-go-with-hours.csv`](fukuoka-city-want-to-go-with-hours.csv) — 表計算ソフトでの確認・編集用

## フィールド（JSON）

`app/src/data/shops.ts` の `Shop` 型に対応するフィールド + 補助情報:

| フィールド | 内容 |
|---|---|
| `name` / `genre` / `area` / `lat` / `lon` / `note` | `Shop` 型と同じ。`note` には定休日情報入り（例:「定休日: 水曜日」） |
| `weekday` / `weekend` | 営業帯。**`{start: [時, 分], end: [時, 分]}` 形式**（`Shop` 型の通算分形式とは異なるので注意） |
| `weekday_r` / `weekend_r` | `r(開時, 開分, 閉時, 閉分)` 文字列（shops.ts 転記用） |
| `hours_status` | `ok`（取得成功）/ `no_hours`（取得不可）/ `mismatch`（店舗一致確認失敗） |
| `hours_by_day` / `raw_hours_aria` | 曜日別の生データ（検証用） |
| `address` / `google_maps_url` | 住所・Google マップ URL（参考情報） |

## データ品質（78 件の内訳）

| hours_status | 件数 | 状態 |
|---|---|---|
| `ok` | 50 | 曜日別営業時間を取得し平日/土日に集約済み |
| `no_hours` | 19 | 営業時間取得不可 → `weekday`/`weekend` は空配列 |
| `mismatch` | 9 | 店舗一致確認失敗（同名他店の混入防止で破棄）→ 空配列 |

## ⚠️ shops.ts へ取り込む前の注意

1. **空配列の 28 件はアプリ上「定休」表示になる** — 使う店だけ手動で営業時間を埋めるか、`hours_status !== 'ok'` の店を除外して取り込む。
2. **平日/土日への集約は機械的** — 曜日ごとに時間が違う店は `Shop` 型の平日/土日 2 区分では正確に表現できない。ただし**曜日別の定休日は `Shop.closedDays`（0=日〜6=土）で表現済み**（`hours_by_day` の closed フラグから生成。2026-06-07 の Codex レビュー指摘対応）。営業時間が曜日で異なるケースのみ残る制約。`hours_by_day` で原データを確認できる。
3. **ジャンルは一部店名からの推定** — 「レストラン」33 件と粗いので、ジャンル選択 UI に使うなら細分化を推奨。
4. **座標は全 78 件あり**（欠損なし）。距離計算（近い順）はそのまま動く。
5. **深夜またぎ営業が 15 店舗・30 営業帯ある（要修正）** — `Shop` 型は深夜またぎ未対応（閉店は 24:00 まで）だが、このデータには閉店が 0:00〜5:00 の店が含まれ、スニペットにも `r(17, 0, 0, 0)` や `r(20, 0, 5, 0)` のまま出力されている。**このまま貼ると `close < open` となり、営業中でも常に閉店扱いになる**。転記時に `r(17, 0, 24, 0)` のように 24:00 で打ち切ること（深夜 0 時以降の営業時間は表現できず切り捨てになる）。
   対象: もも焼き とり昇 / 須崎屋台かじしか / サケサカナ太郎坊 / THE ROAM BAR / 一也ラーメン / 楽しいお酒とごはん genten. / ゴーサイン食堂 / すみ酒場青二才 / 肉肴なおちゃん / Ollie / stand GRETEL / 居酒屋・大衆割烹 文治 / ブラボーラーメン / てっぱんやき屋 福蔵 / 酒場 あらかぶ

## 取り込み方針

1. **重複除去**: ファイル内の自己重複と既存 78 件との重複を 1 件に畳む。
   - **判定は「店名が同一・類似」を必須条件とする**（座標近接だけで畳まない）。同じビル・密集エリアの別店舗を誤って落とさないため。
   - ⚠️ **初回取り込みの不具合と修正（2026-06-12）**: 初回は「同名 **or** 座標約 30m 以内」で畳んでしまい、名前が異なる近接の別店舗 9 件が脱落していた（例: 博多鉄なべ無限餃子＝グリル大宮 JR博多シティ店と26m / 炉ばた 雷橋＝路地裏カレーTiki と18m）。Codex レビュー指摘を受け、座標近接でも**名前が異なる 9 件を復元**（shops.ts 末尾「座標 dedup 復元分」・営業時間は未調査=未確認）。
2. **久留米・小郡 47 件**: 1 店ずつ Web で営業時間を調査して `weekday`/`weekend`（`r()` 形式）・`closedDays`・`tel` を補完。
   - 営業時間を確定できたのは 44 件、確定できなかった 3 件（酎房彩加利・ゲスの極み・豚豚）は空配列＋ note「営業時間未確認」。
   - 元データで緯度が破損していた 9 件（lat≈33.8〜34.1）は Web で正しい座標（久留米≈33.31）に補正（7 件成功）。所在を特定できなかった 2 件（酒寄り処由季・Mano a Mano。いずれも area「螢川町」・lat≈34.03 のまま確定できず）は除外。
3. **福岡市 131 件**: 営業時間は未調査。`weekday`/`weekend` は空・note「営業時間未確認」で取り込み（後日個別に Web 調査して埋める前提・捏造はしない）。
   - うち 2 件（サーラカリーナ福岡店・クボカリー 大楠店）は生データに閉店情報があるため note を「閉店情報あり（要確認）」に変更。
4. **正規化**: `genre` は既存語彙へ寄せて正規化（既存 16 + `日本料理`・`定食` の 2 つを追加）。`area` は「<区/市> <地名>」形式（久留米市・小郡市はそれぞれ「久留米市 ○○」「小郡市 ○○」）。
5. **深夜またぎ**: 閉店が 0 時以降の店は 24:00（`r(.., 24, 0)`）で打ち切り（`close < open` を作らない）。
