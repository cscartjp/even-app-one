# アプリ提出と QA ガイドライン(App Submission & QA Guidelines)

> 原文: https://hub.evenrealities.com/docs/reference/app-submission
> ※ 本ページは公式ドキュメントの非公式な日本語要約です。審査基準は変わる可能性があるため、提出前に必ず原文を確認してください。

Even Hub へアプリを提出する前の必須チェックリスト。

## 1. マニフェスト(app.json)

- `package_id`: リバースドメイン形式、小文字のみ、ハイフン・アンダースコア不可、最低 2 セグメント
- `edition`: `"202601"` 固定
- `name`: 20 文字以下、かつ **「Even」を含めてはいけない**(大文字小文字問わず)
  - 「EvenDoc Reader」「Even Bible」のような名前は公式なりすましとして**自動却下**
  - 公式提携の書面承認がある場合のみ例外
- `version`: `x.y.z`(`v` プレフィックス不可)
- `min_app_version` / `min_sdk_version`: 両方必須(要約時点の下限: `"0.0.10"`)
- `entrypoint`: ビルド出力内に実在するファイル
- `permissions`: `name` + `desc`(1〜300 文字)のオブジェクト配列。network には `whitelist` 必須
- **実際に使っていない権限は却下対象**
- バージョン更新の提出には**空でない変更ログ**が必要

## 2. ストア掲載と視覚アセット

- アイコンは判読可能な設計(なぐり書き・ノイズは不可)。フォアグラウンドとバックグラウンドの両方を供給(null・空は不可)
- **モノクロ / グレースケールのみ。カラーアセットは却下**
- スクリーンショットは実機レンダリングと一致させる(シミュレーターでの撮影を推奨)
- 表示名は `app.json` の `name` とグラス側表示名の両方に一致させる
- 既存アプリのなりすまし、未承認ブランドロゴ、キーワード詰め込みは禁止

## 3. プライバシー

- プライバシーポリシーが**すべての権限**をカバーしていること
- バックエンドのサービスドメインが文書化され、開発者まで追跡可能であること

## 4. 初回起動体験(「黒い画面」は不可)

- 初期設定(都市設定・API キーなど)が必要な場合、グラス起動時に「何をすべきか」を明示するメッセージを表示する。黒い画面のままは不可
- セットアップ内容は `localStorage` API で保持し、再起動のたびに再入力させない
- CORS を正しく設定する(ホワイトリストは CORS の回避ではない →[ネットワーキング](../guides/06-networking.md))

## 5. スマートフォンロック状態での動作

審査チームは**電話ロック + Even Realities アプリがバックグラウンド**の状態でテストする。

- ロック状態でもグラスのアプリが妥当な時間内に描画開始する(無限スピナー不可)
- コアフローがグラス + リング入力だけで完走できる
- すべてのジェスチャーに視覚フィードバック、すべてのボタンに応答がある
- すべての画像がロード完了する
- タイマー等の長時間タスクがロック状態でも正しく完了する
- **2 分間アイドル後も応答可能**(フリーズ・無限ループ・クラッシュ不可)
- 電話のロック解除 → 他アプリ使用 → 再ロック、でグラスのセッションが影響を受けない

## 6. 終了とライフサイクル

- ルートページの**ダブルタップで `bridge.shutDownPageContainer(1)`**(システム終了確認ダイアログ)を呼ぶ
- ルートページでのモード `0`(即時終了)や独自の終了確認 UI は不可
- サイレント終了や「ダブルタップしても何も起きない」アプリは却下対象

参考パターン:

| アプリ | パターン |
|---|---|
| Make15 | ルートのダブルタップでシステムダイアログを直接起動 |
| Chess | ルートのダブルタップで「Exit」項目を含むメニューを開き、選択時に `shutDownPageContainer(1)` |

- グラス側で終了を確認したら、**電話側の WebView も自動的に閉じる**こと(残った WebView は却下対象)
- 終了後、グラスを再起動せずに他のアプリ(Conversate、Navigate 等)を起動できること

ライフサイクルハンドラーの整理:

| イベント | 処理 |
|---|---|
| `FOREGROUND_EXIT_EVENT` (5) | 一時停止・状態のフラッシュ |
| `FOREGROUND_ENTER_EVENT` (4) | 再開 |
| `ABNORMAL_EXIT_EVENT` (6) / `SYSTEM_EXIT_EVENT` (7) | クリーンアップ |

## 7. コンテンツとセーフティ

- 医療診断・金融アドバイス・緊急ルーティング機能は禁止(避けられない場合は法務にエスカレーション)
- 不快なコンテンツ、露骨な表現、NSFW、ヘイト表現は禁止

## 8. 提出前の最終チェック(5 ステップ)

1. `evenhub pack app.json dist -o myapp.ehpk -c` でマニフェスト検証と package_id 確認
2. 電話ロック状態で 5 分間サイドロードし、アプリが生きて応答するか確認
3. ルートのダブルタップでシステム終了ダイアログが出て、WebView が閉じるか確認
4. 終了後、Conversate 等をグラス再起動なしで起動できるか確認
5. プライバシーポリシーが `app.json` の全権限をカバーしているか再確認
