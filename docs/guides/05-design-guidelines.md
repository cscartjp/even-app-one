# UI/UX デザインガイドライン(Design Guidelines)

> 原文: https://hub.evenrealities.com/docs/guides/design-guidelines
> ※ 本ページは公式ドキュメントの非公式な日本語要約です。

## ディスプレイ制約(デザイン観点での再整理)

| 制約 | 内容 |
|---|---|
| キャンバス | 576 × 288 px |
| 色 | 4bit グレースケール(実機では**緑系**で表示される) |
| 背景 | 塗りつぶし不可 |
| コンテナ | 画像 4 つ + その他 8 つまで。入力キャプチャは 1 つのみ |

## アイコン設計の原則

- **ネイティブ解像度で描く**(例: 24×24 で使うなら 24×24 で設計)。縮小スケーリングは避ける
- シンプルに。**シルエットで認識できる**形に
- モニターで見た印象と、実機の緑がかったグレースケール表示は**異なる**。必ず実機(またはシミュレーター)で確認する

## よく使う UI パターン

| パターン | 実装方法 |
|---|---|
| 疑似ボタン | テキストの先頭に `>` を付ける |
| 選択ハイライト | テキストコンテナごとに `borderWidth` を切り替える |
| 複数行レイアウト | テキストコンテナを縦に積む(例: 高さ 96px × 3 段) |
| プログレスバー | Unicode の `━` と `─` を使う |
| ページめくり | テキストを**約 400〜500 文字単位**で事前にページ分割しておく |
